React from Unpkg and user-code from memory

This example is similar to the Simple React example with caching, except that instead of the react code living in the example's index.js, it will create a sort of union filesystem between Unpkg and a set of in-memory files. The in-memory file-system contains the code to render the react components, in the index.js, as well as a package.json file that will tell @velcro/runtime how to resolve modules like `react` and `react-dom`.

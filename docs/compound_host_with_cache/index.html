Simple React (with caching)

This example is similar to the Simple React example with caching, except that instead of the react code living in the example's index.js, it will create a sort of union filesystem between Unpkg and a set of in-memory files. The in-memory file-system contains the code to render the react components, in the index.js, as well as a package.json file that will tell @velcro/runtime how to resolve modules like `react` and `react-dom`.

As an exercise, try modifying the version ranges for react and react-dom and re-running. You will see that the code in the index.js gets versions of its dependencies based on the version ranges found in a package.json file; much like you would get if you had run npm install and then run the code through a bundler.

Here, however, there is no bundler. react and react-dom, as well as their dependencies, are being loaded fully dynamically.
